C++ 核心指南目录

“Use spaces sparingly”

理由

空格太多的话，代码显得很长，很分散。

坏例子 

#include < map >

int main(int argc, char * argv [ ])
{
    // ...
}

好例子 

#include <map>

int main(int argc, char* argv[])
{
    // ...
}

注意

有些 IDE 有自己的设定，会添加一些额外的空格。

如果没有其他更好的想法的话，请遵循这条规则。这条规则是很多人要求添加的。

注意

我们鼓励恰当好处的空格。可以提升可读性。但是不要过多。
